# UI Baseline Matrix (Blocking Tracker)

Last updated: 2026-02-12  
Status: `IN_PROGRESS`

## Purpose
Single execution tracker for full UI standardization before any net-new UI feature work.

## Rules
1. Every UI slice must update this matrix.
2. New UI feature work is blocked until all rows are `DONE` or have an approved documented exception.
3. Status values:
   - `NOT_STARTED`: no baseline work performed.
   - `IN_PROGRESS`: partially migrated; still mixed patterns.
   - `DONE`: token/primitive contract applied and verified.
   - `EXCEPTION`: intentional deviation with rationale + owner.

## Verification Requirements
1. `pnpm check`
2. `pnpm check:settings-smoke` for settings-related slices.
3. Dev-first default: validate slices in `pnpm dev`; reserve `pnpm build` + `pnpm run runserver` for checkpoint slices, explicit server-mode checks, and pre-handoff signoff.
4. Manual smoke on desktop and mobile for touched rows.
